## Overview

Implement a comprehensive error code system for jsql by adding `[dbt:<code>]` error codes to all error messages. The project currently has a partial implementation with 3 error codes (101-103) in `src/dbtyper-error.ts`. This feature will extend this to cover all 361 unique error messages across the codebase.

**Key Insight:** This is a type-level error system where errors are TypeScript types (`SqlParserError<"message">`), not runtime values. Error codes will appear in IDE tooltips and type errors.

**Research Complete:** Planning subagent has completed comprehensive analysis:

- 361 unique error messages identified
- 458 total occurrences across 23 source files
- Categorization scheme proposed (100-999 range)
- Implementation plan with 5 phases

## Technical Challenges

### Challenge 1: Type-Level System

**Problem:** Errors are types, not runtime values
**Impact:** Cannot easily add `[dbt:XXX]` prefix at type level
**Solution:** Keep codes in registry, optionally format in display layer

### Challenge 2: Message Interpolation

**Problem:** Some errors include dynamic values
**Example:** "Expression must be boolean, but has a type X"
**Solution:** Use array format `["part1", "part2"]` for interpolation

### Challenge 3: Similar Contextual Messages

**Problem:** Many similar messages with different contexts
**Example:** "Expected table name in UPDATE/INSERT/DELETE"
**Solution:** Assign unique codes per context for precise identification

### Challenge 4: Duplicate Detection

**Current:** Detects duplicate IDs at compile time
**Issue:** Code 103 has duplicate ID (same as 102)
**Enhancement:** Also detect duplicate messages

### Challenge 5: Test Impact

**Problem:** 1000+ test files reference error messages
**Solution:** Keep original message text unchanged, add code as metadata
