# Implementation Plan - 2026-05-05

## Implementation Order (Dependencies)

### 1. Type System Foundations

- [ ] **Additional PostgreSQL Types** (2-3 hours) - *Nice to have*
  - [ ] **Type Aliases**
    - [ ] timestamptz (alias for "timestamp with time zone")
    - [ ] timetz (alias for "time with time zone")
  - [ ] **Common Types**
    - [ ] serial, bigserial, smallserial (auto-increment)
    - [ ] bytea (binary data)
    - [ ] interval (time intervals)
  - [ ] **Specialized Types**
    - [ ] inet, cidr (network addresses)
    - [ ] tsvector, tsquery (full-text search)
    - [ ] enum types (user-defined)

- [ ] **Type Casts (::type)** (2-3 hours) - *Important*
  - [ ] Parse PostgreSQL :: cast operator
  - [ ] Support for basic types (text, integer, uuid, etc.)
  - [ ] Support for array types (text[], integer[], etc.)
  - [ ] Type validation for casts

### 2. Array Support (builds on Type System)

- [ ] **Array Type Declarations** (2-3 hours) - *Critical*
  - [ ] Support text[], integer[], uuid[], etc. in CREATE TABLE
  - [ ] Type inference for array columns
  - [ ] Array type validation in expressions

- [ ] **Array Operators (<@, =, ||)** (2-3 hours) - *Critical*
  - [ ] `<@` (is contained by) - returns boolean
  - [ ] `=` (array equality) - returns boolean
  - [ ] `||` (array concatenation) - returns array
  - [ ] Type checking for array operations

- [ ] **Array Functions** (2-3 hours) - *Critical*
  - [ ] array_length(array, dimension) - returns integer
  - [ ] array_append(array, element) - returns array
  - [ ] array_prepend(element, array) - returns array
  - [ ] unnest(array) - returns set of elements
  - [ ] Type inference for array functions

- [ ] **ANY() / ALL() / SOME() Operators** (3-4 hours) - *Critical*
  - [ ] Parse ANY(array_expression) syntax
  - [ ] Parse ALL(array_expression) syntax
  - [ ] Parse SOME(array_expression) syntax (alias for ANY)
  - [ ] Type checking: element type must match comparison type
  - [ ] Support with arrays: `WHERE id = ANY(ARRAY[1,2,3])`
  - [ ] Support with subqueries: `WHERE id = ANY(SELECT id FROM table)`

### 3. JOIN Extensions (independent)

- [ ] **RIGHT JOIN** (1-2 hours) - *Critical*
  - [ ] Parse RIGHT JOIN / RIGHT OUTER JOIN syntax
  - [ ] Handle nullable columns from left side
  - [ ] Type checking for JOIN ON conditions

- [ ] **FULL OUTER JOIN** (2-3 hours) - *Critical*
  - [ ] Parse FULL OUTER JOIN syntax
  - [ ] Handle nullable columns from both sides
  - [ ] Type checking for JOIN ON conditions

### 4. Advanced Features (independent)

- [ ] **Custom Operators (<=>)** (2-3 hours) - *Important*
  - [ ] Parse custom operators (PostgreSQL allows custom operators)
  - [ ] Type checking for custom operators
  - [ ] Support for vector distance operators

- [ ] **Window Functions** (3-4 hours) - *Important*
  - [ ] **ROW_NUMBER() OVER (ORDER BY ...)**
    - [ ] Parse window function syntax
    - [ ] Parse OVER clause with ORDER BY
    - [ ] Type inference: ROW_NUMBER() returns integer/bigint
  - [ ] **Other Window Functions** (future)
    - [ ] RANK(), DENSE_RANK()
    - [ ] LAG(), LEAD()
    - [ ] PARTITION BY support
