## SQL Patterns Generated by TypeORM

### 1. Basic WHERE Clauses

**TypeORM Code:**

```typescript
.where("se.project_id = :projectId", { projectId })
.andWhere("se.content_type = :contentType", { contentType })
```

**Generated SQL:**

```sql
WHERE se.project_id = $1 AND se.content_type = $2
```

**dbtyper Support:** ✅ Already supported (parameterized queries)

---

### 2. IN Clauses with Arrays

**TypeORM Code:**

```typescript
.andWhere("e.jira_issue_id IN (:...issueIds)", { issueIds })
.andWhere("meeting.status IN (:...statuses)", { statuses })
```

**Generated SQL:**

```sql
WHERE e.jira_issue_id IN ($1, $2, $3)
WHERE meeting.status IN ($1, $2)
```

**dbtyper Support:** ✅ Already supported (IN with multiple values)

---

### 3. LIKE Pattern Matching

**TypeORM Code:**

```typescript
.andWhere("(LOWER(note.title) LIKE :needle OR LOWER(note.note_raw) LIKE :needle)", { needle })
.andWhere("pin.topic_path LIKE :prefix ESCAPE '\\\\'", { prefix })
```

**Generated SQL:**

```sql
WHERE (LOWER(note.title) LIKE $1 OR LOWER(note.note_raw) LIKE $1)
WHERE pin.topic_path LIKE $1 ESCAPE '\\'
```

**dbtyper Support:**

- ✅ LIKE operator - supported
- ✅ LOWER() function - needs implementation
- ⚠️ ESCAPE clause - needs implementation

### 12. Subqueries in WHERE

**TypeORM Code:**

## Features NOT Found in TypeORM Usage

These features were found in raw SQL but NOT in TypeORM QueryBuilder:

1. **FULL OUTER JOIN** - TypeORM doesn't support this
2. **CROSS JOIN LATERAL** - TypeORM doesn't support LATERAL
3. **Window Functions (ROW_NUMBER() OVER)** - TypeORM doesn't support window functions
4. **CTE (WITH clause)** - TypeORM doesn't support CTEs
5. **INSERT...ON CONFLICT** - TypeORM has `.upsert()` but generates different SQL
6. **RETURNING clause** - TypeORM doesn't expose this in QueryBuilder
7. **Type casts (::)** - TypeORM doesn't generate type casts
8. **ANY() operator** - TypeORM uses IN instead
9. **unnest()** - TypeORM doesn't support set-returning functions
10. **jsonb_array_elements()** - TypeORM doesn't support set-returning functions

**Conclusion:** TypeORM QueryBuilder generates relatively simple SQL. Complex queries use raw SQL via `dataSource.query()`.

## TypeORM `.upsert()` Method

TypeORM has a `.upsert()` method that generates `INSERT...ON CONFLICT`:

**TypeORM Code:**

```typescript
await repository.upsert({ id: "123", name: "John" }, { conflictPaths: ["id"] })
```

**Generated SQL:**

```sql
INSERT INTO users (id, name)
VALUES ($1, $2)
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name
```

**Note:** This is the same as raw SQL `INSERT...ON CONFLICT` found earlier.
