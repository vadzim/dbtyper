# Scalar / `WHERE` expression operations (type-level)

This **typesql** document lists what **`ParseExpressionAST`** / **`ResolveExpressionAST`** (and thus **`ParseWhereExpression`**, typed **`SELECT`** list items, etc.) support, and where we assert **wrong operand types** in tests.

| Category                      | Operation / form                                         | Typed resolve notes                                                                             | Wrong-type test                                                                                                                                                                                                                      |
| ----------------------------- | -------------------------------------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Literal**                   | `true`, `false`                                          | `ExprOk<boolean, "boolean">`                                                                    | —                                                                                                                                                                                                                                    |
| **Literal**                   | `NULL`                                                   | `ExprSqlNull`                                                                                   | `=` to null: `Use IS NULL instead of = null` — `parse-where-expression.test.ts` (`WNullRhs`), `parse-select.test.ts` (`TSelectEqNullErr`)                                                                                            |
| **Literal**                   | string / number                                          | comparison class                                                                                | —                                                                                                                                                                                                                                    |
| **Column / param**            | `alias.col`, `sch.tab.col`, bare `col`, `:name`          | catalog + scope + params                                                                        | unknown column / param tests in `parse-where-expression.test.ts`, `parse-expression.test.ts`                                                                                                                                         |
| **Unary `-`**                 | `-expr`                                                  | **number** only                                                                                 | **`RUnaryMinusText`** — `Unary minus requires a number` — `parse-expression.test.ts`                                                                                                                                                 |
| **Unary `NOT`**               | `not expr`                                               | **boolean** only (not NULL)                                                                     | `NOT requires a boolean operand` — `parse-expression.test.ts` (`RNotNum`), `parse-select.test.ts` (`TSelectNotNumberErr`); `NOT argument must be boolean, not NULL` — `parse-select.test.ts` (`TSelectNotNullErr`)                   |
| **`AND` / `OR`**              | `a and b`, `a or b`                                      | both **boolean**; NULL rejected                                                                 | `AND operands must be boolean` / `OR operands must be boolean` — `parse-select.test.ts` (`TSelectAndNonBoolErr`, `TSelectOrNonBoolErr`); `NULL is not a valid boolean operand` — `parse-select.test.ts` (`TSelectNullAndErr`)        |
| **Compare**                   | `=`, `<>`, `!=`, `<`, `<=`, `>`, `>=`                    | **`SameComparisonClass`** (string / number / boolean / bigint)                                  | `Incompatible types in comparison` — `parse-where-expression.test.ts` (`WBool`), `parse-select.test.ts` (`TSelectCmpTypeErr`, `TSelectCmpStrNumErr`); `parse-update.test.ts` (`UpBadWhere`)                                          |
| **`IS NULL` / `IS NOT NULL`** | `expr is null`, `expr is not null`                       | operand resolves                                                                                | parse errors for bad `IS` tail — `parse-where-expression.test.ts` (`WIsBad`, `WIsNotBad`)                                                                                                                                            |
| **`IN (...)`**                | `expr in (v1, …)`                                        | each element same class as LHS; no NULL elements                                                | `Incompatible types in IN list` — `parse-where-expression.test.ts` (`WInListTypeErr`), `parse-select.test.ts` (`TSelectInListTypeErr`)                                                                                               |
| **`BETWEEN`**                 | `expr between low and high`                              | one class for expr + both bounds; no NULL bounds                                                | `Incompatible types in BETWEEN` — `parse-where-expression.test.ts` (`WBetweenBad` text vs numeric bounds, **`WBetweenNumColStringBounds`** numeric column vs string bounds); **`WBetweenNullBound`** — `NULL not allowed in BETWEEN` |
| **`LIKE` / `ILIKE`**          | `expr like pattern`                                      | **text** on both sides; no NULL                                                                 | **`WLikePatternNonText`** / **`WILikePatternNonText`** — `LIKE pattern must be text`; **`WLikeNullPattern`** — `NULL not allowed in LIKE`; `LIKE left operand must be text` — `parse-where-expression.test.ts` (`WLikeNum`)          |
| **Arithmetic**                | `+`, `-`, `*` (typed value / select / insert cell chain) | **number** × **number** only; no NULL                                                           | **`WArithNumPlusString`**, **`WArithStringPlusNum`**, **`WArithNull`** — `parse-where-expression.test.ts` (`Incompatible types in arithmetic` / `NULL not allowed in arithmetic`)                                                    |
| **`CAST` / `::`**             | `cast(x as t)`, `x::t`                                   | small set of targets (`text`, `integer`, `bigint`, `boolean`, `uuid`, floats/numeric)           | `Invalid cast to integer` — `parse-select.test.ts` (`TSelectCastIntErr`, `TSelectPgCastBoolIntErr`); **`WCastTextToInteger`** (`text` column → `integer`) — `parse-where-expression.test.ts`                                         |
| **`CASE`** searched           | `case when … then … [else …] end`                        | `WHEN` **boolean**; branches merge with same comparison class                                   | `CASE WHEN must be boolean` — `parse-where-expression.test.ts` (`WCaseWhenNotBool`); `Incompatible types in CASE` — `WCaseIncompat`                                                                                                  |
| **`CASE`** simple             | `case expr when … then … [else …] end`                   | each **`WHEN`** value **`=`**-compatible with **`expr`**; **`THEN`/`ELSE`** merge like searched | `Incompatible types in comparison` — `parse-where-expression.test.ts` (`WCaseSimpleWhenMismatch`); same **`Incompatible types in CASE`** for branch mismatch (`WCaseIncompat` pattern applies to both forms)                         |
| **Boolean context**           | whole `WHERE` / root                                     | must resolve to **boolean**                                                                     | `Expression must be boolean` — `parse-expression.test.ts` (`WNonBoolRoot`)                                                                                                                                                           |

**Parse-only** (not in the table above): `IN` / `BETWEEN` / `CASE` delimiter and paren structure, `Invalid comparison operator`, `Expected NULL after IS`, `Unbalanced parentheses`, `Unexpected token`, etc. — covered in `parse-where-expression.test.ts` and `parse-select.test.ts` as syntax / shape errors.
